手機感測器互動測試平台

本平台用於展示如何透過瀏覽器 Web API 存取手機內建的各種感測器和功能，並實現互動效果。

專案說明

純手機或平板，以網站形式來做到感測器互動是完全可行的，並且是現代網頁技術的趨勢。關鍵在於利用手機和平板內建的各種感測器和功能，這些都可以透過瀏覽器的 Web API (Application Programming Interface) 來存取。

純手機/平板網站的優勢：

  • 無需額外硬體： 不用教具包，只要有手機或平板就能玩。
  • 普惠性： 大大降低了參與門檻，更多小朋友能體驗。
  • 即時互動： 直接透過手機的感測器來觸發網頁內容，反應迅速。
  • 真實世界應用： 強調手機/平板本身就是一個充滿感測器和科技的裝置。

可測試的感測器與功能：

  • 麥克風 (Microphone) 測試
  • 加速度計/陀螺儀 (Accelerometer/Gyroscope) 測試
  • 環境光感測器 (Ambient Light Sensor) 測試
  • 相機 (Camera) 測試
  • 觸控螢幕 (Touch Screen) 測試
  • 地理位置 (Geolocation) 測試

重要考量與限制：

  • 權限請求： 這些敏感權限都需要使用者明確同意。網站設計時需要提供清晰的說明。
  • 瀏覽器兼容性： 不同瀏覽器對某些 Web API 的支援程度可能不同。
  • 安全性： 確保網站的程式碼安全，不濫用取得的權限。
  • 背景運行： 某些感測器在背景運行會消耗電量。
  • 精確度： 手機內建感測器的精確度可能不如專業的獨立感測器。
  • 無法直接控制 GPIO： 手機網站無法像 Micro:bit 或 Arduino 那樣直接控制電路板上的引腳。
